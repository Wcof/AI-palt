# 核心 Agent 业务落地规划文档：AI EHS 平台能力矩阵

**核心产品愿景**：打造场景驱动、懂标准、能办事的 AI 安全助理。通过将底层核心能力封装为 MCP (Model Context Protocol) 服务，并在前端挂载于不同的业务 Agent，实现从“隐患发现”、“业务阻断”、“管理决策”到“人员赋能”的全链条闭环。

---

## 1. 隐患排查 Agent（现场人员的“超级外脑”）

* **角色定位**：辅助一线安全员与网格员，提升隐患识别的专业度与效率。
* **业务场景**：日常现场巡检、全员随手拍隐患上报。
* **解决痛点**：新手看不出专业隐患；手工填报隐患单繁琐，导致现场人员不愿上报。
* **底层能力依赖 (MCP)**：
  * 多模态图像识别 API（支持最多 20 张批量识别）。
  * 企业 EHS 组织架构与工单流转 API。
* **PM 关键设计与避坑指南**：
  * **Top-K 确认与负反馈飞轮**：AI 输出高置信度的 Top 3 选项供用户点选。若用户选择“其他”，则自动收集为负反馈数据，用于私有化模型微调。
  * **环境叠加研判（连环追问）**：识别到单一隐患（如漏水）时，AI 需主动追问上下文（如“周边是否有配电设施？”），引导用户进行动态风险升级评估。

---

## 2. 特殊作业管控 Agent（高危场景的“铁面审核官”）

* **角色定位**：辅助安全主管，死守高危作业（动火、受限空间、高处等）的安全红线。
* **业务场景**：特殊作业票证的智能申请、合规性前置审核与签发。
* **解决痛点**：规避“体外循环”，解决纸质/传统电子票证的“形式审核”问题（如气体检测数据造假、监护人脱岗）。
* **底层能力依赖 (MCP)**：
  * OCR 视觉解析 API（精准读取气体检测仪表盘读数、特种作业证件信息）。
  * 逻辑校验规则引擎（时间戳与定位交叉验证）。
* **PM 关键设计与避坑指南**：
  * **硬逻辑前置阻断（Hard Rules）**：AI 必须具备“时间差”与“空间差”的逻辑校验能力。如气体检测仪拍照时间超过规范要求，Agent 必须直接冻结流程。
  * **尽职免责边界**：Agent 的输出必须是“风险提示”与“强烈驳回建议”，绝不代行“同意签发”的权力，最终签字动作必须由人类完成。

---

## 3. 法规与合规 Agent（企业总监的“便携智库”）

* **角色定位**：充当企业 EHS 专家的合规翻译官。
* **业务场景**：查阅国家/行业标准（GB/AQ）、编写应急预案、解读模棱两可的合规争议。
* **解决痛点**：法规浩如烟海且晦涩难懂，传统搜索无法结合具体业务场景给出执行清单。
* **底层能力依赖 (MCP)**：
  * RAG 向量检索 API（基于私有化部署的标准库和企业制度库）。
  * 外部联网搜索 API（校验最新法规废改立动态）。
* **PM 关键设计与避坑指南**：
  * **零幻觉容忍（Zero Hallucination）**：强制 Agent 回答必须带有精确的溯源引用（如 `[来源：《动火作业规范》第3页]`）。绝不允许自行发散编造标准。
  * **场景化法条翻译**：将晦涩的条文直接翻译成现场人员听得懂的“施工前准备 Action List”。

---

## 4. AI 决策辅助 Agent（管理层的“数据参谋”）

* **角色定位**：面向厂长、EHS 总监的数据洞察与趋势预测助手。
* **业务场景**：月度安全例会数据复盘、高频隐患趋势研判、安全预算规划。
* **解决痛点**：业务系统数据孤岛严重，传统 Dashboard 报表固定死板，领导无法随时通过自然语言进行下钻查询。
* **底层能力依赖 (MCP)**：
  * NL2SQL API（自然语言转 SQL，直接对底层隐患、作业票数据库进行动态查询）。
  * 图表动态渲染 API（生成 Echarts / 柱状图 / 热力图）。
* **PM 关键设计与避坑指南**：
  * **数据权限隔离（RBAC）**：必须严格校验提问者的身份。车间主任只能查询本车间的数据，防止越权穿透引发管理问题。
  * **可解释的预测**：当 AI 给出“下个月动火作业风险较高”的预测时，必须强制附带数据支撑（如：“由于下个月进入设备大修期，历史同期违章率上升 15%”），避免玄学决策。

---

## 5. AI 智能题库 Agent（企业级 EHS 的“专属私教”）

* **角色定位**：对标 Gemini 的启发式学习助手，替代传统的刻板考试与培训系统。
* **业务场景**：新员工三级教育、承包商入厂资质摸底、特种作业考证冲刺辅导。
* **解决痛点**：传统培训纯属背题和应付差事；员工做错题后不知错因，无法形成真正的安全潜意识。
* **底层能力依赖 (MCP)**：
  * 动态多模态考题生成 API。
  * 苏格拉底式对话（Socratic Dialog）引擎。
  * 用户知识追踪（Knowledge Tracing）数据库。
* **PM 关键设计与避坑指南**：
  * **启发式辅导（Tutoring Mode）**：摒弃直接给答案的做法。用户答错时，通过引导式提问（如：“你再观察一下图片里的风向标，如果泄漏，往哪个方向跑最安全？”），让用户自行推导出结论。
  * **多模态情景出题**：抛弃纯文字题干，直接发送现场设备图片或违规作业动图，要求用户“圈出违规点并说明理由”。
  * **千人千面的雷达图**：系统后台建立动态能力模型，自动针对用户的“盲区知识点”在后续测验中提高出题权重。
