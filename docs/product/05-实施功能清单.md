# 实施功能清单

排序原则：先做商业化、账号、多租户和鉴权底座，再做助手质量闭环、工具/OCR、模型与审计，最后做小程序、容灾和体验增强。已删除或已具备的能力不再作为新增项保留在本表。

| 编号 | 功能域 | 功能项 | 状态 | 需要做什么 | 前端开发入口 | 后端/API 依赖 | 验收点 | 优先级 |
| --- | --- | --- | --- | --- | --- | --- | --- | --- |
| F-01 | 商业化授权 | 功能包 / 能力包定义 | 新增 | 定义基础版、标准版、专业版、企业版能力编码和功能包关系 | `src/views/newAI/settings/systemManage` 或新增授权配置页 | 能力编码、套餐、客户授权接口 | 能看到某客户已开通/未开通能力 | P0 |
| F-02 | 商业化授权 | License 客户端校验 | 新增 | 客户端读取 License 状态并在入口展示授权状态 | newAI 全局状态、助手入口、管理页入口 | License 查询、有效期、能力范围 | 过期/未授权时入口和接口都能拦截 | P0 |
| F-03 | 商业化授权 | API 层功能开关 | 新增 | 前端只展示状态，最终开关落在 API 层 | API 请求拦截、错误处理、付费提示组件 | 功能开关接口、标准错误码 | 未开通功能返回明确能力包提示 | P0 |
| F-04 | 商业化授权 | Agent 调用鉴权 | 需增强 | Agent 调用 Skill、知识库、数据源、模型前统一走 API 鉴权 | `src/views/newAI/assistant`、`src/components/assistant` | Agent 调用鉴权、能力编码、请求 ID | 未授权不直接失败，能提示付费开通 | P0 |

| F-07 | 多租户 | 多租户后端底座 | 非 Web 主体 | 后端补齐租户上下文、数据隔离、租户级鉴权、租户审计，前端只消费当前租户上下文 | Web 侧仅展示当前客户 / 项目空间，不新增复杂租户配置页面 | tenantId/projectId、数据权限、租户审计 | 不同租户不能互相访问知识库、Agent、数据源、日志和用量 | P0 |
| F-08 | 多租户 | 多租户隔离 | 需增强 | 知识库、数据源、授权范围隔离 | newAI 设置页、知识库/Agent 管理页 | 租户/客户空间、数据权限 | 不同客户只能看到自己的能力和数据 | P0 |
| F-09 | 助手体验 | 回答反馈入口 | 新增 | AI 回答下方增加点赞、点踩、纠错、错误类型 | `src/views/newAI/assistant/components/GeneralChatPanel.vue` | 反馈提交、反馈类型、日志 ID | 专家能提交反馈并进入反馈列表待处理 | P0 |

| F-12 | 助手体验 | 换行统一 | 需增强 | 明确 `Shift+Enter`、`Alt+Enter`、`Ctrl+Enter` 等换行/发送规则，并统一输入、输出、复制、Markdown、移动端展示 | `ComposerShell.vue`、消息渲染组件 | 无强后端依赖，需约定模型上下文格式 | Web/移动端/复制内容换行一致 | P0 |

| F-14 | 问题反馈 | 反馈列表 | 新增 | 管理问答、识图、NL2SQL、会议纪要、工具失败案例，形成业务专家反馈处置入口 | 新增 `src/views/newAI/settings/samplePool` 或问题反馈同级页面 | 反馈 CRUD、分类、状态流转 | 反馈可进入统一列表并分类处理 | P0 |
