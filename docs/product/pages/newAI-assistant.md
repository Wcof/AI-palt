# AI 助手

## 页面基础信息
- 路由：`/newAI/assistant`
- 兼容路由：`/ai-app`
- 页面定位：默认首页工作台

## 页面目标
- 承接新会话、Agent 选择、消息发送、上传/OCR、专家共创反馈、上下文打包和模型切换。

## 页面结构
- 左侧主菜单与近期会话栏
- 中间聊天工作台
- 顶部模型切换
- 顶部会话上下文打包入口
- 底部反问补充面板
- 消息操作区

## 字段说明
- Agent 名称
- 模型来源、模型名称、版本、耗时、日志 ID
- 上传文件状态

## 操作说明
- 发送消息
- 点击左侧“新会话”时创建新的通用助手会话
- 点击左侧“智能问答”进入默认问答会话
- 点击左侧“隐患识别”进入隐患识图会话
- 点击左侧“智能文书”进入文档编写会话
- 点击左侧“更多”打开更多功能浮层，进入 AI 智能体、AI 技能、AI 知识库和开发工作台
- 对歧义问题进行补充回答
- 切换 Agent
- 上传图片并识图
- 用户问题 Hover 后可通过 icon 复制、修改
- 修改问题后从当前问题重新开始，清空后续问答
- AI 回答在气泡下方固定展示复制、重新生成、点赞、点踩 icon 操作
- 点赞直接进入专家共创
- 点踩必须选择类型并填写原因；同一条 AI 回复再次点踩时回显上次内容并覆盖更新
- 从顶部入口打包当前会话上下文
- 登录弹窗仅提供密码登录和快捷登录两种方式，不再展示顶部“登录 / 注册”切换
- 快捷登录扫码成功后进入基础信息补全，个人用户填写名称、手机号、密码和确认密码；企业用户填写企业名称、手机号、所在行业、企业规模、密码和确认密码
- 基础信息补全后直接进入登录态并关闭弹窗，不新增独立注册路由
- 会话列表中的单条会话提供更多操作，可执行删除、导出和重命名

## 交互规则
- Agent 固定为：通用助手、智能问数、文档编写、会议纪要、知识库问答、隐患识图。
- 点击左侧“新会话”时，默认进入未聚焦的通用助手会话，不继承上一条会话的 Agent 类型。
- 左侧主菜单常用入口顺序固定为：新会话、智能问答、隐患识别、智能文书、更多。
- 左侧近期会话列表的单条会话 Hover 后显示三点更多操作，点击后展示删除、导出和重命名。
- 删除会话从本地会话列表移除；导出会话下载当前会话 JSON；重命名会更新会话标题与更新时间。
- “更多”采用侧向浮层面板，不再使用内嵌折叠列表。
- “更多功能”浮层包含标题、说明、关闭按钮和 2 列入口卡片；入口固定为 AI 智能体、AI 技能、AI 知识库、开发工作台。
- 点击隐患识别、智能文书、会议纪要、文档编写时，当前工作台切换到对应 Agent。
- 当前原型中，用户每次发送文本问题后，AI 都会先进入 mock 反问流程，再继续输出正式回答。
- 上传后生成 OCR 结果，OCR 结果进入上下文和样本池。
- 用户消息 Hover 才显示复制、修改；复制和修改均采用 icon 形式，不显示文字按钮。
- AI 回复操作按钮固定显示在气泡下方，采用 icon 形式常显。
- “修改” 会把原问题回填到输入框，发送后截断该问题之后的全部消息并重启当前轮会话。
- “修改” 态输入框需比普通消息气泡更宽，使用蓝色渐变输入区域承载原问题内容。
- “重新生成” 会按该回答对应的上一条用户问题重新生成，并清空该轮之后的全部消息。
- 反问补充面板以覆盖层形式挂在底部输入容器内部，固定提供 3 个建议补充选项和 1 个“其他补充信息”输入框。
- 建议补充项按单列纵向排列，每行只展示一条选项。
- 反问补充面板出现时遮住原问题输入框和底部发送操作区，避免同时出现两个输入入口。
- 用户点击建议选项或输入其他补充信息后，当前补充内容会作为新的用户消息进入会话，再继续生成正式回答。
- 点赞、点踩都进入“专家共创”数据池；点踩需要记录反馈类型和原因。
- 点踩按 `会话 ID + 消息 ID + dislike` 唯一覆盖保存，重复点踩不新增多条记录，而是更新原记录。
- 登录弹窗仅提供密码登录 / 快捷登录切换，不再提供独立注册页签。
- 密码登录仅做本地 mock 校验，需填写账号和密码，密码不持久化。
- 快捷登录展示二维码；点击模拟扫码成功后，本地生成待完善资料的二维码用户。
- 扫码成功后必须补全基础信息：个人用户填写名称、手机号、密码、确认密码；企业用户填写企业名称、手机号、所在行业、企业规模、密码、确认密码。
- 基础信息补全后写入 `user` store 并标记资料已完善；密码仅做前端一致性校验，不持久化；不再使用短信验证码、邮箱验证码或验证码倒计时。

## 状态流转
- 空态 → 用户发问 → AI 反问中 → 用户补充 → 发送中 → 已返回
- 上传中 → 已上传 / 上传失败 / 可重试

## 异常场景
- 未授权时弹付费提示。
- mock 返回失败时显示错误信息和重试入口。

## 权限规则
- OCR / 隐患识图、模型切换受能力包控制。

## 数据规则
- 会话、专家共创反馈、上下文包、上传、OCR、模型配置全部使用 Pinia persisted state。
- 账号登录和快捷登录均为本地 mock 流程；密码仅用于前端校验，不入库；二维码用户扫码后需补全基础资料。

## 验收标准
- 可以发送 AI 消息。
- 左侧菜单按智能问答、隐患识别、智能文书、更多组织，并可切换对应页面或 Agent。
- 发送文本问题后会先进入反问补充流程。
- 可以通过 3 个建议选项或“其他补充信息”继续补全问题。
- 可以切换 Agent。
- 未授权功能触发标准提示。
- 可以修改历史问题并从该条重新开始。
- 可以对单条 AI 回答执行重新生成。
- 点赞、点踩可以进入专家共创，重复点踩会回显并更新单条回复的既有反馈。
- 可以打包会话上下文。
- 可以在登录弹窗中使用密码登录或快捷登录；快捷登录扫码后能按个人 / 企业补全基础信息并自动进入登录态。
