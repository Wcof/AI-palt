# 会议纪要操作条整改说明

## 本次调整范围

本次仅针对「AI 应用 / 会议纪要」页面底部操作区域进行整改，对齐用户截图红框中的音频播放器式操作条。

## 已完成调整

1. 将原来的「启动 / 暂停 / 停止 / 状态说明」分散式操作区，调整为播放器式底部控制条。
2. 控制条结构调整为：
   - 左侧回到开头按钮；
   - 主播放 / 暂停圆形按钮；
   - 快进 / 录音中停止确认按钮；
   - 中间波形进度区；
   - 下方展示当前时间与总时长；
   - 右侧倍速入口；
   - 右侧音频设置入口。
3. 会议纪要场景下隐藏原底部 Agent 标签、更多能力、深度思考等通用对话操作，避免和音频操作条混杂。
4. 保留录音中暂停、继续、停止二次确认的基本逻辑：
   - 录音中点击主按钮为暂停；
   - 暂停后点击主按钮为继续；
   - 录音中/暂停时点击第三个小按钮为停止，第一次触发确认，第二次确认停止；
   - 非录音状态下主按钮用于播放/开始。
5. 增加响应式处理，小屏下隐藏倍速和设置，优先保留核心播放与波形区域。
